> ⚠️ **Disclaimer**: This is an **experimental** modular CLI toolset. The core dynamic routing architecture (`main.py`) and all built-in modules (`commands/*.py`) were 100% generated by **Gemini 3.1 Pro**.

A zero-dependency Python CLI toolset built with AST dynamic routing, **designed to be fully extended by AI.**

PF Tools features zero-latency startup, supports subcommands and aliases, and allows highly flexible hot-pluggable extensions without modifying the core router.

## 📁 Core Structure

The repository uses a minimalist, flat, and modular design:
```text
PF_Tools/
├── pt.bat            # Global CMD entry point (Add this directory to system PATH)
├── main.py           # Core command router & AST-driven dynamic parser
├── prompt.txt        # Core asset: AI module development specification (Prompt)
└── commands/         # Hot-pluggable plugin directory
    ├── compare.py    # Fast file comparison module
    ├── ffmpeg.py     # Hardware-accelerated media transcoding module
    └── get_frame.py  # Precise video frame extraction module
```

## 🛠️ Built-in Test Modules

Three test modules are currently built-in. Run `pt help` or `pt <command> -h` for full documentation.

| Command | Alias | Description |
| :--- | :--- | :--- |
| `pt compare` | `comp` | **Fast File Compare**: Size & dynamic sampling based comparison, with optional full binary deep comparison. |
| `pt ffmpeg` | `ff` | **Interactive Media Compressor**: Video/image compression utilizing FFmpeg CUDA hardware acceleration. |
| `pt get_frame` | `gf` | **Frame Extractor**: Extract specific video frames using positive/negative indexing (e.g., `-1` for the last frame). |

*Note: The `ffmpeg` and `get_frame` modules require `ffmpeg` (with `hevc_nvenc` support) configured in system environment variables.*

## 🚀 How to Add New Features (AI-Driven)

The core advantage of this project is its **"AI-programmability"**. 
To add new features, **no manual coding or modification to `main.py` is required**:

1. Open `prompt.txt` in the root directory.
2. Copy the "Sub-module Development Specification" to your LLM.
3. Describe your need, e.g., *"Based on the specs, write an audio extraction module named extract_audio, alias ea."*
4. Save the AI-generated single Python file directly into the `commands/` directory. The system will automatically route it and generate Help docs.

## 🛡️ Security & Review Mechanism (Design Philosophy)

This project strictly adheres to "Complete Transparency, No Black Boxes":

- **Mandatory Source Review**: All extension commands must be plain-text `.py` scripts. Obfuscated Python code (e.g., PyArmor) in the `commands/` directory is strictly prohibited.
- **Cross-language & Pre-compiled Modules**: If you use C/C++, Rust, Go, etc., compiled into dynamic libraries (`.pyd`, `.so`, `.dll`) or executables for performance, **you must open-source this underlying code**. Closed-source black-box distribution is not allowed.
- **Calling External Independent Programs**: Calling existing closed-source/independent tools in the user's system (like `fc`, `ffmpeg`) via `subprocess` is allowed, as this is normal Inter-Process Communication (IPC).
- **GPL Copyleft Statement**: Under the GPLv3 license, any distribution of this project's core logic tightly bound (e.g., in-process linking) with closed-source components without providing source code constitutes an infringement.
