> ⚠️ **声明**: 本项目为一个**实验性**的模块化命令行工具集。项目的核心动态路由架构（`main.py`）及所有内置子功能模块（`commands/*.py`）均完全由 **Gemini 3.1 Pro** 编写生成。

一个零依赖、基于 AST 动态路由的 Python CLI 工具集，**专为全凭 AI 扩展功能而设计**。

PF Tools 实现了子模块的毫秒级零延迟动态路由，支持子命令、短指令别名，并允许高度自由的热插拔扩展。

## 📁 核心目录结构

本仓库采用极简的扁平化与模块化设计：
```text
PF_Tools/
├── pt.bat            # 全局 CMD 统一入口（需将所在目录添加至系统 PATH）
├── main.py           # 核心指令路由与动态解析器 (AST 驱动)
├── prompt.txt        # 核心扩展资产：AI 子模块开发规范 (Prompt)
└── commands/         # 插件化功能存放区 (支持热插拔)
    ├── compare.py    # 快速文件比对模块
    ├── ffmpeg.py     # 视频/图片硬件加速转码模块
    └── get_frame.py  # 视频极速抽帧模块
```

## 🛠️ 内置测试功能

本项目目前内置了三个基于核心架构的测试性功能模块，详情可运行 `pt help` 或 `pt <command> -h` 查看完整使用文档。

| 指令 | 别名 | 功能描述 |
| :--- | :--- | :--- |
| `pt compare` | `comp` | **极速文件比对**：基于文件大小与动态分段抽样的快速比对，可选全量二进制深度比对。 |
| `pt ffmpeg` | `ff` | **交互式媒体压缩**：交互式视频/图片压缩与转换 (利用 FFmpeg CUDA 硬件加速)。支持单文件极速预设，及高级模式下的目录遍历、自动化重命名等。 |
| `pt get_frame` | `gf` | **视频精准抽帧**：使用 FFmpeg 提取视频特定帧，支持正倒数索引（如 `-1` 提取最后一帧）及智能输出路径识别。 |

*注：`ffmpeg` 与 `get_frame` 模块需系统环境中已配置好 `ffmpeg` (包含 `hevc_nvenc` 硬件加速支持)。*

## 🚀 如何扩展新功能

本项目的核心优势在于**“面向 AI 的可编程性”**。

如果你需要增加任何新功能，**无需手动编写代码，也无需修改 `main.py`**：

1. 打开仓库根目录下的 `prompt.txt`。
2. 将其中的“子模块开发规范”复制给你的大语言模型。
3. 输入你的需求，例如：“*基于规范，写一个提取音频的模块，指令名 extract_audio，别名 ea*”。
4. 将 AI 生成的单文件直接保存到 `commands/` 目录下即可。系统会自动将其接入全局路由并生成 Help 文档。

本项目坚持“完全透明、拒绝黑盒”的开源精神。为了保证主程序及所有运行在用户机器上的指令都是安全可控的，我们对子模块有以下约束：

- **基础要求（强制审查）**：所有的扩展指令核心逻辑必须可通过源码进行审查。严禁在 `commands/` 目录中引入经过混淆处理的 Python 代码（如使用 PyArmor）。
- **跨语言与预编译模块**：如果你的子功能为了性能，需要使用其他语言（如 C/C++, Rust, Go 等）编译为动态链接库（`.pyd`, `.so`, `.dll`）或独立的可执行文件供 Python 引导调用，**你必须同时开源这部分底层代码**，不可作为一个无源码的黑盒闭源分发。
- **调用外部独立程序**：允许通过 `subprocess` 等方式调用用户系统中已有的闭源或独立的第三方工具（如 `fc`, `ffmpeg` 等），这属于正常的进程间通信，不违反透明性原则。
- **GPL 传染性声明**：根据 GPLv3 协议，任何将本项目核心逻辑与无法提供源码的闭源组件强绑定（如进程内链接）并对外分发的行为，均构成侵权。
